## Introduction

복잡한 흐름을 단순하게 구조화하고, 함께 이해할 수 있는 결과물을 만드는 것을 좋아합니다.<br>
2년 동안 SI 회사에서 일하며, 코드란 단순히 기능을 만드는 도구가 아니라 동료들과 소통하고 협업하는 매개체라는 점을 느꼈습니다.

혼자서는 해결할 수 없는 문제를 함께 풀어가는 과정의 중심에는 코드가 있다고 생각합니다. <br>
기능을 구현하는 것만큼 어떻게 구현할지, 그리고 왜 그렇게 구현하는지를 중요하게 생각합니다. <br>
쉽게 포기하지 않고 차근차근 파고들며, 처음 접하는 문제도 작게 쪼개고 맥락을 이해하며 해결해 나가는 과정을 즐깁니다. <br>

좋은 코드는 시간을 지나도 이해할 수 있는 문서이자, 팀 전체의 기준점이 된다고 생각합니다. <br>
그 기준을 함께 고민하며 더 나은 방향을 찾아가는 개발자가 되고자 합니다.

## Projects

### 🧺 Sokuri
- 사용자가 URL만 입력하면 실제 가방 사이즈를 기반으로 물건을 넣어볼 수 있는 IOS 애플리케이션입니다.<br>
- 기술 스택: React Native, Swift, Three.js, FastAPI, Zustand, Python
- [README](https://github.com/sokuri-org/sokuri-crawler/tree/feature/docs)

### 🐞Snapbug
- React 애플리케이션의 상태 변화와 DOM을 함께 저장해, 디버깅과 상태 공유를 쉽게 만들어주는 타임트래블 디버깅 도구입니다.
- 기술 스택: JavaScript, React, Vite, Vercel, npm, Node.js
- [README](https://github.com/snap-bug/snap-bug-client/tree/feature/readme)
